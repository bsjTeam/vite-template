关于commit提交规范

```js
// feat：新功能（feature）
// fix：修补bug
// docs：文档（documentation）
// style： 格式（不影响代码运行的变动）
// refactor：重构（即不是新增功能，也不是修改bug的代码变动）
// test：增加测试
// chore：构建过程或辅助工具的变动
// revert：回滚
// build：打包
// ci：持续集成
// perf：性能优化
// init：初始化
```
